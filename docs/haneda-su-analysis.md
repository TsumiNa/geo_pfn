# Haneda 真实数据实验分析报告

> 数据说明见 [`haneda-su-dataset.md`](haneda-su-dataset.md)，实验设计见
> [`haneda-experiment-plan.md`](haneda-experiment-plan.md)。原始数字在
> `results/haneda/{ablation,imputation,context}.json`（每臂每折指标 + 配置回显），
> 逐行预测在 `results/haneda/predictions/`（不入库，`geo_pfn.haneda.run` 可再生）。
> 运行环境：Apple Silicon MPS，总耗时 95 分钟（2026-07-08）。

## 0. 结论速览

| 问题 | 答案 |
|---|---|
| **Q1** miniPFN vs TabPFN v2（均零微调） | **v2 全面胜出**：分箱分类任务上配对差 +2.7 ~ +4.1 pt（全部显著）。miniPFN 只与逻辑回归打平（~0.646），且 **MNAR 增强训练（cells）相对 vanilla 在真实数据上没有任何增益**（三个特征集上 +0.15 / +0.58 / −0.59 pt，均在噪声内） |
| **Q2** 地质数据的贡献 | **有真实但次要的增益**：便宜土工参数是大头（v2-reg RMSE 17.49→14.92，配对 −2.57±0.38）；土类 +粒径合计再降 −0.67±0.14（约 4.5% 相对改善），其中土类 −0.30±0.09、粒径 −0.37±0.12。只有地质没有土工参数（LSG）明显不如只有土工参数（LC） |
| **Q3** imputation 在真实 MNAR 上的效果 | **合成实验的"不要预填补"效应在真实数据上消失**：native ≈ mean（v2-reg 配对差 +0.009±0.086）。可操作的新发现：**手动拼接 0/1 缺失指示列让 v2-reg 显著获益（−0.21±0.08 RMSE）**，而 KNN 填补显著有害（+0.25±0.10）。物理重建 e 有小幅正向但不显著（−0.07±0.05） |

**给 geo_pfn 的实用配方（当前最优）**：TabPFN v2 回归 + 全部特征（位置 +
便宜土工 + 土类 + 粒径）+ 缺失指示列 ≈ **RMSE 14.0 kN/m²、MAE 9.5、R² 0.79**，
对比仅用深度的线性基线（RMSE 19.0、R² 0.62）误差下降约 26%。

## 1. 协议回顾（一句话）

240 个钻孔 5 折整孔留出（GroupKFold, seed 42），回归目标 Su 原值，分类目标为
训练折四分位分箱（4 类，迁就 miniPFN 分类头上限）；所有臂共享同一套折，
比较一律用逐折配对差值（mean ± SEM，n=5 折）。

## 3. Q1：miniPFN vs TabPFN v2

1. **v2 的领先真实且随特征复杂度增大**。配对差（v2-clf − mini-cells）：
   LC +3.4±0.7 pt，LCS +2.7±0.7 pt，LCSG +4.1±0.8 pt——全部 ≥4 倍 SEM。
   与上一轮 wine@60% 缺失的差距（7.7 pt）方向一致、量级相近，
   "综合能力 v2 明显更强"的结论在真实表格上成立。
2. **miniPFN 的真实水位 = 逻辑回归**。mini-cells 在所有特征集上与
   linear 贴合（差 <1 pt），说明 1.6M 合成先验模型迁移到真实数据后
   能捕捉主要线性结构，但对 hgbt/v2 能吃到的非线性（+2~4 pt）无能为力。
   纯位置特征（L）时所有模型打平 0.63——深度信号谁都能学会，
   区分度全部来自土工参数与地质特征的非线性利用。
3. **MNAR 增强训练在真实数据上没有兑现**（核心发现的边界条件）：
   mini-cells − mini-vanilla 配对差 = +0.15±0.41 / +0.58±0.54 /
   −0.59±0.66 pt（LC/LCS/LCSG），无一显著、方向不稳。
   原因在 EDA 中已可预见：真实缺失模式经深度控制后只携带
   约 3–4 kN/m²（≈0.2 SEM 折算）的额外信号，且以钻孔为块
   （测试孔要么整孔有粒径要么整孔没有），远弱于合成 MNAR 里
   "缺失率随类别翻倍"的构造信号。**上一轮的结论
   "MNAR 利用能力 = 训练分布"仍然成立，但它兑现为真实收益的前提是
   数据里确实存在足够强的行级缺失-标签耦合；本数据集不满足。**
4. **上下文规模扫描**：128×16 集成（0.647）≥ 512×4（0.643）≥
   1024×2（0.640）。超出训练分布（60–160 行）的长上下文单打
   不如分布内短上下文多次集成——mini 模型外推行数的能力有限，
   集成是正确用法。

## 4. Q2：地质特征的贡献

以 v2-reg 的逐折配对差衡量（RMSE，负 = 改善）：

| 增量 | 配对差 | 显著性 |
|---|---|---|
| L → LC（+6 便宜土工） | **−2.57 ± 0.38** | 6.7×SEM，大头 |
| LC → LCS（+土类） | −0.30 ± 0.09 | 3.4×SEM，小而显著 |
| LCS → LCSG（+粒径） | −0.37 ± 0.12 | 3.2×SEM，小而显著 |
| LC → LCSG（地质合计） | −0.67 ± 0.14 | 5.0×SEM |

- **层级结论**：便宜土工参数 ≫ 地质 ≫ 纯位置。地质单独上阵（LSG 15.59）
  不如便宜土工单独上阵（LC 14.92）。
- hgbt 呈同样格局（LCSG−LC = −0.68±0.21），但**土类对 hgbt 无增益**
  （−0.07±0.10, ns）而对 v2 有——v2 的 categorical 通道比树模型的
  整数码分裂更会用这个 10 类标签。
- 粒径列 52.5% 缺失仍然贡献了显著改善：对"是否值得做颗分试验"的
  勘察决策，这是一个正面证据（哪怕一半钻孔没有，混着用也有效）。

## 5. Q3：imputation 在真实块状 MNAR 上的重新评估

v2-reg @ LCSG 的逐折配对差（vs native NaN；负 = 比 native 好）：

| 策略 | RMSE 配对差 | 判定 |
|---|---|---|
| mean | +0.009 ± 0.086 | **打平**——合成实验中"mean 处处更差"在真实数据上消失 |
| knn | **+0.250 ± 0.096** | **显著更差**——编造貌似合理的值误导强模型 |
| mean+indicator | **−0.212 ± 0.084** | **显著更好**——待做队列 #1 得到正面答案 |
| physics-e | −0.069 ± 0.051 | 方向正确但不显著 |

分类侧（v2-clf、mini-cells、linear、hgbt）所有策略差异全部 |Δ|≤0.8 pt、
无稳定方向——分箱后的粗粒度目标对填补方式不敏感。

解读三条：

1. **"不要预填补"需要限定条件**。合成先验里特征强相关，mean 填补
   破坏相关结构、损失巨大；真实数据缺失列（粒径、e）与其余特征的
   互信息本来就低，mean 填补无信息可毁。上一轮结论修正为：
   **预填补最多无益（mean）甚至有害（KNN），native NaN 是安全默认值，
   但其优势幅度取决于特征相关结构**。
2. **KNN 的失败模式值得注意**：它比 mean 差 0.25（而在 linear 上
   反而略好 −0.08±0.04）。对 v2 这种会精细利用特征值的模型，
   "看起来真实的假值"比"明显是均值的占位值"更具误导性。
3. **指示列是免费午餐**：−0.21 RMSE（≈1.5%）虽小但 2.5×SEM 且与
   EDA 预测的信号量级吻合（深度控制后缺失指示携带几 kN/m² 信息）。
   结合上一轮"v2 native 完全不利用缺失模式"的发现，机制自洽：
   **v2 自身不读缺失模式，把它显式变成特征列它就能读**。
   这就是零训练成本弥补 MNAR 缺口的办法，推荐进入默认配方。

## 6. 稳健性检查

- **Su 离群值（max 365.9）敏感性**：cap 在 p99.5（147.8）或剔除后，
  所有模型 RMSE 一致下降 ~0.9（pooled 口径），**排名与全部结论不变**。
  例：v2-reg LCSG 14.36 → capped 13.39 → 剔除 13.10（pooled）。
- **有序视角**：within-1-bin 准确率所有模型 ≥0.96（v2 最高 0.975）——
  没有模型犯"跨两档"的严重错误，4 类分箱任务的差异集中在相邻档位分辨率。
- 消融与填补两组实验中 v2-reg @ LCSG/native 独立复算两次
  （14.248 vs 14.248），管线自洽。

## 7. 对 geo_pfn 路线的含义

1. **短期基线已立**：v2-reg + 全特征 + 指示列（RMSE≈14.0，R²≈0.79）是
   后续一切自研模型必须打败的线。它零训练、零微调、许可安全。
2. **mini 路线的瓶颈排序更新**：真实数据上 mini ≈ logreg，说明当前
   合成先验缺的不是 MNAR 增强（本轮证伪其真实收益），而是
   (a) 回归头（分箱丢信息，v2-reg 对 v2-clf 的优势侧面印证）、
   (b) 更宽先验（>10 特征、类别变量、更真实的特征相关结构）、
   (c) 行数外推（上下文扫描显示 >160 行即衰减）。
   待做队列中"回归头 + 更宽先验"应提到 MNAR 机制泛化实验之前。
3. **勘察决策角度**：粒径（颗分试验）混合使用有显著增益，
   即使一半钻孔没有；物理公式重建 e 无损且方向正面，应默认启用。
4. **微调方向（方案 C）的预期收益**上限可以量化了：v2 与 hgbt 的差距
   （~1 RMSE）加上指示列增益（0.2）给出"更懂缺失/更懂领域的 v2"
   大致 1–1.5 RMSE 的空间；相对地，把 mini 拉到 v2 水平需要跨越
   ~2.4 RMSE 等价的能力差距——微调 v2 仍是性价比更高的下一步。

## 8. 后续实验一：方案 C——微调 TabPFN v2（负结果，已关闭）

用官方 `tabpfn.finetuning` 在每折训练孔上微调 v2 回归器（强制 v2 权重规避
v2.5 许可问题；`geo_pfn.haneda.finetune`），逐折与零微调配对（LCSG）：

| 验证方式 | native 配对差 | mean+ind 配对差 |
|---|---|---|
| 行随机（官方默认） | +0.07 ± 0.17（5 折） | +0.18 ± 0.16（5 折） |
| 钻孔分组（修复版） | +0.07 ± 0.08（5 折） | +0.19 ± 0.20（4 折，提前终止） |

**判定：两种验证方式下微调均无收益**（正 = 比零微调差）。行随机验证还
引入大幅折间波动（fold 0 曾 +0.66）——训练孔的行泄漏进验证集使早停失灵、
产生对训练钻孔的空间过拟合；钻孔分组验证消除了波动但也只是回到打平。
预期的 1–1.5 RMSE 微调空间在 2.8k 行单场地数据上不能兑现，方案 C 关闭，
资源转向 TabICL 路线与自研先验（方案 D）。

## 9. 后续实验二：TabICLv2 基线（新的最优，权重可商用）

TabICLv2（soda-inria，代码与权重均 BSD-3）在相同折上配对评估
（`results/haneda/tabicl.json`，CPU 即可，~90 s/fit）：

| 任务 | TabICLv2 | TabPFN v2 | 配对差 |
|---|---|---|---|
| 回归 RMSE（native） | **13.86±0.68** | 14.25±0.76 | −0.39±0.20（5 折全胜） |
| 回归 R²（native） | **0.797** | 0.786 | +0.011±0.005 |
| 分类 acc（native） | **0.707±0.012** | 0.688±0.016 | +1.9±0.6 pt |
| 分类 acc（mean+ind） | 0.708 | 0.687 | +2.1±0.4 pt |

指示列对 TabICL 无增益（回归 +0.011±0.088）——其行压缩表征已隐式覆盖
缺失模式携带的批次信息。**新的推荐配方：TabICLv2 + 原始 NaN 输入
（RMSE 13.86 / R² 0.80），许可无任何商用限制。**

## 10. 残差结构分析（A5/i.i.d. 假设的量化，指向下一步）

对 v2-reg 残差的分解（详见 session 记录，2026-07-09）：

- 钻孔级 ICC = 0.195，孔偏差 ±8.6 kPa；oracle 消除后 RMSE 14.36→12.44
  （**−1.9，大于此前一切干预之和**）；孔内 lag-1 残差自相关 0.256;
- 孔偏差与残差剖面形状的**空间相关均为零**（邻孔 <150 m 亦然），
  即空间平滑信号已被模型经 X/Y 特征吃尽；岩性序列本身空间相干
  （邻孔一致率 0.374 vs 远孔 0.24），但其可转移内容已包含在输入特征中;
- **推论**：新钻孔场景（本报告协议）的孔偏差不可收割；
  **孔内补全（infill）场景可全额收割 −1.9**——上下文放入同孔已测行即可，
  零训练成本。infill 评估是下一步性价比最高的实验；层序 warping 对齐
  （DTW 检验）是场景 A 残存的最后假设。

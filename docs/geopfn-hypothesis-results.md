# geo-PFN 架构假设检验：钻孔相似性利用（2026-07-09）

> 检验用户的架构级猜想：一个**显式"列 embed → 行比对"的两段式 PFN**
> （`geo_pfn.geopfn`），在多孔 site 先验（`geo_pfn.geoprior`）上训练，能学出
> 一个 geo 专属的行度量，比通用 per-cell ICL（TabPFN v2）和 TabICL 更会利用
> 稀疏锚点的跨孔迁移。全部在真实 Haneda 数据上评估，与已提交的 v2/TabICL/HGBT
> anchor 结果直接配对可比。原始数字：`results/haneda/anchor_geopfn_*.json`。

## 0. 判决速览

| 子命题 | 判定 |
|---|---|
| 两段式架构能学会利用锚点（增益随 k 单调增长） | ✅ **成立** |
| 锚点利用能力 competitive with the best | ✅ **成立**（Su-LCSG k=5 增益 −2.07 为全场最大；Wn 增益也最大） |
| 纯空间+深度冷启动追平 v2 | ✅ **成立**（Su-L holdout 17.4 ≈ v2 17.1） |
| **绝对精度超过 v2** | ❌ **不成立**——v2 在所有设定的绝对 RMSE 都更好 |

**一句话**：猜想的**机制部分被真实数据证实**——两段式地质度量确实学会了用
钻孔/深度相似性做迁移，锚点增益 competitive、局部最优，且在"仅坐标+深度预测
虚拟钻孔"这个愿景核心场景上追平 v2。但 2.0M 参数、纯合成先验、8000 步的模型
**绝对精度还赢不了 7M 的 v2**，差距来自诊断清楚的"先验真实度"缺口（见 §4）。

## 1. 建了什么

- **geo-SCM 多孔 site 先验**（`geo_pfn.geoprior`）：一次采样一个 site（若干口孔），
  共享 site 级 SCM + 随坐标平滑的空间场（近孔相似），孔内深度轴分层、层内平滑
  层间跳变（标定自实测），混合单/多孔，上下文=稠密邻居+稀疏锚点。
- **两段式 geo-PFN**（`geo_pfn.geopfn`，2.0M 参数）：① 列编码器（沿样本轴注意力，
  仅上下文行为 key）② CLS 行压缩 ③ 行间注意力（query 行 attend context 行 =
  学习度量/软 kNN）+ bar-distribution 回归头（值+不确定性）。
- 训练：8000 步、MPS、~1.5→0.3 it/s（per-cell 的 3.6×，效率优势兑现）。
  rmse_norm（归一化）从基线 1.29 降到 0.70，loss 4.16→2.03，正常收敛。

## 2. 核心结果：锚点迁移（Su，LCSG 全特征）

配对 anchor−holdout RMSE（负=锚点有帮助），5 折 mean±SEM：

| 模型 | k=1 增益 | k=3 增益 | k=5 增益 | k=5 绝对(holdout→anchor) |
|---|---|---|---|---|
| **geo-PFN** | −0.50 | −1.40 | **−2.07±0.30** | 22.7 → 20.6 |
| v2-reg | −0.57 | −1.57 | −1.92±0.46 | 14.4 → **12.5** |
| TabICL | −0.11 | −0.94 | −1.43±0.57 | 14.2 → 12.8 |
| HGBT | −0.33 | −0.73 | −1.31±0.39 | 15.5 → 14.2 |

- **锚点增益**：geo-PFN 的 k=5 增益 −2.07 是**全场最大**，且增益随 k 增长最快
  ——架构确实学会了"多几个锚点就精准迁移"，这是猜想机制的直接证据。
- **绝对 RMSE**：geo-PFN（20.6）远差于 v2（12.5）——冷启动/绝对精度弱。

## 3. 零信息场景（Su，L：仅坐标+深度）——愿景核心

| 模型 | holdout（冷启动） | k=5 增益 | k=5 绝对 |
|---|---|---|---|
| **geo-PFN** | **17.4** | −2.53 | 15.8 |
| v2-reg | 17.1 | −3.70 | 14.1 |
| HGBT | 17.3 | −2.75 | 14.8 |

- **冷启动 holdout：geo-PFN 17.4 ≈ v2 17.1**——在"只有坐标+深度、预测全新虚拟
  钻孔"这个愿景最核心的场景,geo-PFN **绝对精度追平了 v2**。合成先验把**深度
  结构**学得很好、能迁移到真实数据。
- 加锚点后 v2 反超（14.1 vs 15.8）——v2 的锚点利用在 L 上更强。

## 4. 诊断：绝对精度差距 = "先验真实度"缺口（可修）

关键反直觉现象:**geo-PFN 在 L(17.4)比在 LCSG(22.2)预测更好**——加了真实
廉价土工特征反而变差。对照 Wn 目标更明显:

| 目标 | v2 holdout | geo-PFN holdout |
|---|---|---|
| Su / L(仅位置) | 17.1 | **17.4（追平）** |
| Su / LCSG(+土工) | 13.8 | 22.2（差） |
| Wn / LCSG | 4.1 | 8.7（差） |

**诊断**：geo-PFN 的合成先验教会了它**深度/空间结构**(迁移好),但**没教会
真实廉价土工特征之间的关系**(如 Wn↔LL↔e 的物理耦合,v2 靠它把 Wn 预测到
RMSE 4.1 / R²0.96)。合成先验的特征分布 ≠ 真实地质物理。这是**先验真实度**
问题,不是架构或度量机制问题——用更真的先验(掺入真实特征相关结构)或在真实
数据上微调即可补,路径清楚。

## 5. 多属性能力（Wn 含水率）

geo-PFN 也能预测其他连续目标（masked-cell 多属性愿景可行）：Wn holdout RMSE
~8.3（std~20，R²≈0.83），锚点增益随 k 增长到 −0.83。但与基线对比再次印证
先验真实度缺口：

| 模型 | Wn holdout RMSE | 备注 |
|---|---|---|
| v2-reg | **4.0**（R²≈0.96） | 靠 LL/e 等土工特征的物理耦合，几乎完美 |
| HGBT | 4.7 | |
| geo-PFN | 8.3 | 先验没学会这些物理耦合 |

v2 把 Wn 预测到 R²0.96、锚点几乎无增益（已饱和）；geo-PFN 8.3、锚点仍有
可观增益（未饱和，还有信息没榨干）。

## 6. 土层分类：锚点内插的最强体现

预测未测深度的岩性（soil_B02，~10 类），特征=位置+土工+粒径，v2-clf/HGBT 基线：

| 模型 | 冷启动 acc | +3 锚点 acc | 增益 |
|---|---|---|---|
| v2-clf | 0.481±0.019 | **0.835±0.016** | **+0.35** |
| HGBT | 0.491±0.017 | 0.747±0.026 | +0.26 |

**这是"锚点内插"机制最强的一次证据**：冷启动只有 ~48%（土型难从物性反推），
但给 3 个散布深度的土型标签，v2 直接跳到 **83.5%**。原因是**岩性在深度上是
分段常数（层状）**——知道几个层界标签，中间深度几乎就是内插填空。这与 geo-SCM
先验"层内平缓、层间跳变"的设计假设完全一致，也解释了为什么散布锚点对分段目标
的价值远大于对连续 Su 的价值。（geo-PFN 分类头是待办项 §7.4，此处用基线量化可
预测性。）

## 7. 结论与下一步

**猜想判决:机制成立,绝对精度待补。** 两段式 geo 度量学会了利用钻孔相似性
(锚点增益全场最优、冷启动追平 v2),验证了用户对"metric learning over
borehole similarity"的核心直觉。但要在绝对精度上超过 v2,还需要:

1. **先验真实度(最高优先级)**:让合成先验的廉价土工特征关系逼近真实物理
   ——这是 §4 诊断出的主缺口,直接决定绝对精度;
2. **规模**:2.0M → 更大(v2 是 7M),配更多训练步;
3. **真实数据微调**:在 Haneda 训练折上微调(方案 C 对 geo-PFN 重做,geo-PFN
   权重自有、无许可约束);
4. **分类头**:加土型分类头(§6),补齐 masked-cell 双头愿景。

**已验证的资产**:多孔 site 先验 + 两段式架构 + bar-dist 回归头 + 真实数据
评估管线全部跑通,迭代成本低。geo-PFN 在"仅坐标+深度预测虚拟钻孔 + 不确定性"
这个愿景核心场景上已 competitive,是一个可迭代的正向起点。

## 8. 可视化

预测 vs 真实值的直观对比见 [`geopfn-viz.html`](geopfn-viz.html)（校准散点 +
四口钻孔的 Su-深度剖面）：仅位置时两模型散点云等宽（打平），加土工后 v2 贴对角线、

## 9. 复现

```bash
uv run pytest src/geo_pfn/geopfn/ src/geo_pfn/geoprior/   # 38 测试
uv run python -m geo_pfn.geopfn.train --steps 8000        # MPS ~5-6h
uv run python -m geo_pfn.geopfn.eval_anchor --feature-set LCSG --target Su
uv run python -m geo_pfn.geopfn.eval_anchor --feature-set L --target Su
uv run python -m geo_pfn.haneda.anchor --feature-set L --target Su   # 基线
uv run python -m geo_pfn.haneda.anchor_soil --k-anchors 3            # 土层分类
```

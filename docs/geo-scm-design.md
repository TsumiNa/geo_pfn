# geo-SCM 先验设计（钻孔生成器）

> 方案 D 的核心。目标：预训练时**一次采样一整口钻孔的 table**，让 PFN 学到
> "位置 + 廉价土工 + 深度 + 地层结构 → 昂贵目标 + 不确定性"的推断能力。
> 设计由 `data/pilot_Su_domain_block_mod4Liu.csv` 的实测结构标定（见 §1）。
> 实现：`src/geo_pfn/geoprior/prior.py`；验证：`geo_prior_test.py` 断言
> 采样统计落在实测区间内。

## 1. 实测标定（geo-SCM 必须复现的结构）

| 结构 | 实测值 | 在先验里的角色 |
|---|---|---|
| 每孔试样数 | 中位 14、最多 37 | `max_rows` 提到 **300**（留扩展余量） |
| 每孔土层数 | 中位 3、均值 3.2、最多 9 | 分层数 `K ~ 1+Poisson`，clip [1,9] |
| 层厚 | 中位 8.9 m、p90 28 m | 边界按深度随机切 |
| 单试样薄层占比 | 40% | 边界密度允许薄层 |
| **跨界/层内跳变比** | **1.4–3.5×**（均 ~2.5×） | 层内平滑步长 vs 边界跳变的核心比例 |
| 采样间距 | 中位 1.5 m | 深度轴 `z` 增量 ~Exp(1.5) |
| 土型-深度关系 | 有倾向、不严格单调 | 层土型采样带深度偏置 |
| corr(depth, Su) | −0.79 | 深度是潜场的主输入 |
| 特征数 | 19 列 | `max_features ≈ 28`（19×1.5） |

## 2. 生成过程（一次产出一口钻孔 = batch 中一个 table）

四层结构，从"世界"到"观测"，与真实钻探过程一一对应：

### L0 — 钻孔上下文（"世界"）
- **位置潜变量 `u`**（低维）：整口孔共享的常量，映射到观测的 `X, Y` 列，
  并对整条剖面施加一个孔级偏移（复现实测 ICC 0.195 的钻孔级偏差）。
- **深度轴 `z`**：`n_rows` 个深度，增量 ~Exp(mean 1.5 m)、累积、排序、
  从近地表起。`n_rows ∈ [min_rows, 300]`。

### L1 — 地层（分段）
- 层数 `K = 1 + Poisson(λ≈2.2)`，clip [1, 9]。
- 在 `z` 上随机切 `K-1` 个边界 → 每行落入一个层 `layer(z)`（分段常数函数）。
- 每层采一个**土型标签**（离散），带轻微深度偏置（深层偏向某些型）。
  → 映射到观测的 `soil_code` 列（分类特征）。

### L2 — 潜属性场（**分阶段平滑：层内平缓、跨界突变**）
这是设计的核心，直接实现你指出的结构：
- 维护一个潜状态向量 `s(z)`（d 维），沿深度演化：
  - **层内**：`s` 做低方差随机游走（或深度的平滑函数），步长 `σ_within`；
  - **跨层边界**：`s` 叠加一次**大跳变**，幅度 `σ_jump = ρ · σ_within`，
    `ρ ~ U[1.5, 3.5]`（标定自实测跳变比）。
- 每层另有一个**层级潜签名**（该层的"基线"），边界处切换——这让同一土型
  的层有相似基线、不同土型的层基线不同。

### L3 — SCM 映射（潜场 → 观测特征 + 目标）
- `[s(z), depth, 层签名, u]` 过一个**随机 MLP-SCM**（复用 miniPFN/TabICL 的
  随机权重因果图）生成节点池。
- **因果分层（关键）**：
  - 浅/中层节点 → **廉价土工列**（Wn, LL, PL, Gs, ρt, e —— 上游/总在场）；
  - **深层节点 → 目标 Su**（下游/待预测）。
  这既复刻 SCM 的"目标=深节点"，又匹配物理（强度是含水率/塑性/密度的果）。
- `depth` 本身是一个观测列且是潜场主输入 → 自然产生 corr(depth,Su)≈−0.8。
- 特征数补齐到 `n_features`（≤28），多余列取节点池中未用节点（扩展余量）。

### L4 — 观测层（测量与记录）
- 每列随机仿射（scale/shift）+ 轻观测噪声（qu 试验噪声、偶然不确定性来源）。
- **缺失**：
  - 块状列缺失（整列对整孔缺失，概率标定自粒径 52.5% / e 22%）；
  - **廉价在场、昂贵掩码**的结构化增强 —— 训练时高频练习"从上游+邻居
    填下游"，把先验对齐到部署时的推理模式（§4）。

## 3. 与三份现成先验的关系

- **骨架抄 miniPFN/TabICL**：L3 的随机 MLP-SCM + 节点采样直接复用；
- **新增的 geo 结构**：L0 孔级潜变量、L1 分层、L2 分段平滑+跳变 —— 这三层是
  所有通用先验都没有的（它们假设行 i.i.d.，见分析文档 §10 的 A5）；
- **Tree-SCM 可选混入**（TabICL 的 `_tree_scm.py`）：土型→属性的阈值型
  关系用树生成器更自然，作为 L3 的一个分支按比例混合。

## 4. 掩码增强对齐推理（geo 的结构化不对称）

通用 PFN 对"哪列缺失"对称；geo 用例不对称——廉价列几乎总在、昂贵列总要预测。
训练增强因此**偏置**向：以高概率 mask 下游目标列（Su/qu）、保留上游廉价列，
并可 mask 部分行的全部属性（模拟稀疏钻孔的"锚点"场景，见 anchor 实验）。
这让预训练直接练习两种部署模式：① 行内（廉价→昂贵）② 跨行（邻孔迁移）。

## 5. 不确定性（愿景的核心交付）

geo-SCM 的两类噪声对应两类不确定性，PFN 预测分布同时捕获：
- **偶然**（不可约）：L4 观测噪声 + 层内随机游走的固有散布；
- **认知**（可约）：上下文里邻孔/锚点越多 → 后验越尖 → 预测区间越窄。
校准性由"先验采样分布 = 训练分布"保证——**前提是本设计的联合结构够真**。
下一步须把**区间覆盖率**升为一等评估指标（分析文档路线图第 3 级）。

## 7. 验证（实现后必须通过）

采样 N 口合成钻孔，断言统计落在实测区间：
1. 层数分布中位 ≈ 3、范围 [1, 9]；
2. 跨界/层内属性跳变比 ∈ [1.4, 3.5]；
3. corr(depth, target) 显著为负；
4. 廉价列与目标的互信息 > 纯位置列与目标（因果分层生效）；
5. 目标列的孔级偏差方差 > 0（L0 生效）。
可视化：抽几口孔画 `属性 vs 深度` 剖面，肉眼确认"层内平缓、跨界跳"。

## 8. 表的组织：多孔 site（决策：混合、偏多孔）

**为什么必须多孔**（anchor 实验的实证依据，2026-07-09）：散布深度的稀疏
锚点带来随 k 增长的迁移增益（TabICL k=5 达 −1.43 RMSE），但**通用 ICL
不比 HGBT 重训练强**（−1.43 vs −1.31，SEM 内）——现成模型的行度量没针对
地质相似性调过，只吃到"更多数据"、没吃到"认出正确邻居"。geo-PFN 的差异化
价值就是**更好的学习度量**，而它只能从"多口孔同时在场"的训练表里学到；
单孔表里坐标是常量，跨孔迁移不可学。故：

- 一张训练表 = 一个 **site**（一个小区域的若干口孔），共享：
  - 一套 **site 级 SCM**（该区域的因果律，使不同孔可比）；
  - 一个**随坐标平滑的空间场**，把每口孔的潜场整体抬升/压低 → 近孔相似；
- **混合采样**（`p_single`）：一部分表单孔（练孔内深度/层结构与廉价→目标），
  多数表多孔（练跨孔迁移）——单孔是"只有一口孔的 site"的特例；
- **上下文掩码 = anchor 场景**：site 内部分孔为"稠密邻居"（整孔观测），
  部分为"稀疏目标"（仅留 k 个散布深度的锚点，其余为查询）。
  这让预训练直接练习部署时的推理：从稠密邻居 + 目标孔的稀疏锚点预测其余。
  （单孔表则按 `train_frac` 随机切上下文/查询。）

实现：`src/geo_pfn/geoprior/site.py`；`GeoBatch` 增加逐行 `context_mask`。
验证：同 site 内近孔的目标相关性 > 远孔（空间场生效）。

## 9. 模型侧方向（下一步，用户指定）

优化 miniPFN 时**借鉴 TabICL 的分阶段相互注意力**：先把列数据 embed（列级
编码器沿行聚合分布），再在行向量层面比对——从架构层面直接模拟我们讨论的
"钻孔/试样相似性"机制（第三段 ≈ 学习度量空间里的软 kNN，正是 anchor 迁移
所需）。相比 miniPFN 现有的 per-cell 交替注意力，这让"行作为单位的相似性"
成为一等建模对象，与多孔 site 先验的训练目标对齐。它还顺带解决 R=300 的
效率瓶颈（per-cell 注意力在 300 行上已明显变慢，实测 ~2 s/步）。

### 9.1 双头设计（尺寸均可配，不写死 4）

masked-cell 形态下模型要能预测**任意被 mask 的列**，而列分两类，各接一个头：

- **连续列**（depth 派生、廉价土工、Su/qu）→ **回归头（bar-distribution）**：
  把连续目标离散成 ~数百个 bin，输出 bin 上的概率分布。**既出点估计
  （分布均值/中位）又出 uncertainty（分布宽度）**——正是愿景要的"值 +
  不确定性"，也是能与 TabICL/v2 同台比 RMSE 的前提。官方 `tab_pfn_src`
  有现成 bar-distribution 实现可借鉴。
- **分类列**（土层类型 soil_B02）→ **分类头，类数 = 土型词表大小**：
  现有 MiniPFN 的 `max_classes=4` **远不够**——真实数据 ~19 数字码 / 24
  字母码，先验 `n_soil_types` 默认 10。分类头维度须取 `n_soil_types`
  （评估真实数据时取真实词表大小），做成可配。土型本身既是可预测目标
  （推断未测深度的岩性）也是输入特征。

当前分箱版（4 类回归的临时替身）**只用于验证先验能否驱动训练收敛**，
不能与 TabICL/v2 比 RMSE（单位不一致 + 4 箱量化天花板 RMSE≈11.8）。
真正的可比版本 = **多孔 site 先验 + 回归头 + 尺寸正确的分类头 +
两段式注意力**。

## 10. 廉价特征的先验真实性（geo-realistic block）

**动机。** 假设检验（`docs/geopfn-hypothesis-results.md`）发现 geo-PFN 在
**加了真实廉价土工特征后反而变差**（LCSG 整孔 holdout RMSE 22.9 > 只用坐标+深度
的 L 19.3）。诊断：不是训练不足，而是**先验里的廉价特征分布与真实分布相反**。
§2 的随机 MLP-SCM 让每个特征都（a）信息量大、（b）彼此近似独立；真实 Haneda
的廉价特征恰好相反。

**实测结构**（`fit_stats.py` 从 pilot 拟合，写入 `haneda_stats.json`）：

| 事实 | 实测 | 含义 |
|---|---|---|
| 深度单独解释 log Su | R²=0.564 | 深度是主导驱动 |
| +6 个廉价特征 | R²=0.683（**+0.12**）| 廉价特征只带**微弱**增量 |
| 廉价簇 PC1 方差占比 | 0.765 | 6 列是**一个高度共线的簇**（"软硬/含水"轴）|
| PC1 对 Su 的偏相关\|深度 | **+0.019** | 主导轴与深度冗余，**几乎不含 Su 信号** |
| PC3/PC4 偏相关\|深度 | −0.35 / +0.31（方差仅 8%）| 信号藏在**低方差的特征差**里 |

即：**主导的高方差共线轴与深度冗余、几乎无 Su 信号；真正的 +0.12 R² 藏在
低方差的特征间差异里。** 随机 MLP 先验把这套关系教反了——它让模型去信任
高方差特征方向，正是这份数据里最错的直觉。

**geo-realistic block**（`realistic.py`，`sample_features` 按 `p_geo_realistic`
概率整表启用）复现上述两条事实,由 geo-SCM 场里已有的两个平滑潜因子驱动:

- `soft`——主导"软硬/含水"轴,沿真实 PC1 方向加载廉价簇(Wn/LL/PL/e 同向、
  rho_t 反向)、带深度趋势(故与深度冗余)、对目标贡献≈0。**76% 方差、~0 信号**。
- `signal`——一个**小幅度**的特征差轴,喂给目标一个**弱**耦合项。特征方差低,
  却是"深度之外 +0.12 R²"的来源(扮演 PC3/PC4 的角色)。

每个特征再经**真实列的经验分位函数**映射,使边缘分布也对齐(近常数的 Gs、
零膨胀的粒径等)。逐 site 抖动加载/差轴/目标系数,使其成为一个**律的分布**
(family)而非单一映射——模型仍须从 context 读出每个 site 的耦合。生成的
合成表复现:PC1≈0.71、深度 R²≈0.63、增量≈+0.12、PC1 偏相关\|深度≈0,与实测
吻合。剩余一半表仍走随机 MLP 分支(`p_geo_realistic=0.5`)保留广度。

验证:`realistic_test.py`(共线性、弱耦合、主导轴无系统信号、边缘范围、确定性)。
A/B 检验:同预算(12k 步、seed 0)训练新旧先验各一版,对比 L 与 LCSG 的整孔
holdout——成功判据 = LCSG 不再劣于 L(先验真实性缺口被填上)。

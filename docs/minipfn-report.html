§1 · 实验设计

三个模型，唯一变量是训练时的缺失增强

三个模型共享全部结构与训练配置：per-cell token 双轴注意力 transformer（d=128、6 层、4 头、1.6M 参数，复刻 TabPFN v2 的布局）、同一个随机 MLP/SCM 合成任务先验（内建每任务 U[0,10%] 的 MCAR 单元缺失）、12,000 步 × batch 32 ≈ 38.4 万个合成任务、相同随机种子。唯一的差别是每个训练批次上施加的缺失腐蚀：

模型训练增强训练命令
vanilla无额外增强（只有先验内建的 ≤10% 随机单元缺失）--augmentation none
columns50% 任务：随机 1..F/2 个整列在测试行置 NaN--augmentation columns
cells70% 任务：任务级缺失率 u∼U[0,0.6]，每行独立缺失率 ∼U[0,2u]，作用于全部行；其中 30% 任务为标签相关缺失（MNAR）--augmentation cells

评估时所有策略在完全相同的任务与相同的缺失位置上对比（配对设计，种子固定），合成结果为 400 任务均值 ± 标准误。

真实基线：原版 TabPFN v2 checkpoint（7M 参数、约 1.3 亿合成数据集预训练、n_estimators=8 集成）通过 eval_tabpfn.py 重放同一随机序列参战——各表中 logreg 行数值逐位一致证明了掩码级配对。注意：合成任务采样自 mini-PFN 自己的先验，对三个 mini 模型是分布内、对 v2 是分布外（主场优势）；跨模型的横向水平请以真实数据和「native − mean-impute 差值」为准。

§2 · 缺失场景

两种缺失的形态不同，难点也不同

下图每个小格是数据表中的一个单元格；■ 观测到，▨ 缺失（NaN）。横线上方是 train 上下文行，下方是待预测的 test 行。

场景一对应「测试时某些传感器/字段整体不可用」；场景二对应问卷调查——1000 份问卷每份作答完整度不同，train 和 test 都不完整。MNAR 是场景二的加难版：哪道题没答本身与标签相关（例如收入高的人更倾向跳过收入题），此时缺失位置携带真实信号。

§3 · 策略与基线

每种策略到底做了什么

五种策略分两个家族。彩色 = mini-PFN（权重冻结，单次前向，零重训练），灰色 = 经典流程（每个任务重新拟合一套填补器 + 分类器）。

mini-PFN · native NaN 零改造 · 推荐

NaN 直接进模型。每个缺失单元格编码为 Linear([0, flag=1]) + 列嵌入——模型同时知道「这里缺了」和「缺的是第 j 列」，可在 in-context 中对缺失做近似贝叶斯边缘化，也可把缺失模式当特征。

mini-PFN · mean-impute 消融：隐瞒缺失

先用 train 行的列均值把所有 NaN 填掉再喂模型，缺失 flag 恒为 0——模型被「隐瞒」了缺失事实。与 native 的差值 = 缺失指示通道的净贡献。

TabPFN v2 · native / mean-impute 真实基线

原版 v2 权重逐任务 fit(X_train, y_train) → predict(X_test)。native 直接喂 NaN（v2 编码器自带缺失指示 + train 均值填补）；mean-impute 先填补再喂、隐藏缺失模式。两者之差 = v2 对缺失信号的利用程度。

logreg · mean-impute

SimpleImputer(strategy="mean") → StandardScaler → LogisticRegression，逐任务在该任务的 train 行上拟合。缺失单元格一律替换为所在列的 train 均值——最常见也最粗糙的做法，缺失位置信息被完全抹掉。

logreg · mean+indicator 经典最强对照

同上，但 add_indicator=True：为每个含缺失的列额外拼接一个 0/1 指示列（特征数近乎翻倍），让线性模型也能利用「缺没缺」这个信号。这是统计学里 1970 年代的 missing-indicator 方法，也是 MNAR 场景下最应该赢的经典方案。

logreg · knn-impute

KNNImputer(n_neighbors=5)：对每个缺失单元格，用 NaN 感知欧氏距离（只在双方都观测到的维度上算距离、再按维度数缩放）在 train 行里找 5 个最近邻行，取它们在该列的均值填入；test 行同样对着 train 行找邻居。比均值填补聪明——它利用了特征间的相关性做「局部回归式」填补，但仍然丢弃缺失位置信号。

场景一（整列缺失）专属的两种策略

  • mini-PFN · drop-context：把缺失列从 train 上下文中也删掉、在缩减后的特征空间里重新推断。对 in-context 学习器这是免费的（fit 无梯度）；但对逐行缺失场景不适用——不能只为某一行删一列。
  • logreg · retrain：只用幸存列重新训练——「特征变了就重训」的教科书答案，代价是每种缺失组合都要重训一次。
§4 · 结果 · 场景一

整列缺失：平滑退化，全面优于经典基线

  • 缺失一半的列仍保有大部分精度（0.704 → 0.616/0.627），且每个缺失水平都超过为该缺失组合专门重训的 logistic 回归（0.599）。
  • drop-context 略优于 nan-fill（50% 时 0.627 vs 0.616）：整列缺失时把列从上下文一起删掉稍微更干净，两条零改造路线都可用。
  • mean-impute 是明确的反面教材：隐瞒缺失让准确率崩到 0.557——比告诉模型「这里缺了」低 6 个百分点。

§5 · 结果 · 场景二（问卷式）

逐行随机缺失：native NaN 全程领先，且低比例训练可泛化到高比例

  • native NaN 在每个缺失率上都是最优策略，60% 缺失时领先最强填补基线约 3 个百分点（0.568 vs 0.539）。
  • cells 与 vanilla 两条 native 曲线几乎重合（相差 ≤0.4pt）：vanilla 训练时只见过 ≤10% 的随机缺失，却能平滑外推到 60%。随机缺失（MCAR）的鲁棒性来自「见过缺失」这件事本身，不需要见过高比例。
  • 真实基线 TabPFN v2（紫色）落在 mini 模型之下、logreg 之上——这些任务对 v2 是分布外（主场劣势），横向水平不代表其真实实力；真正有效的信息是它自身的 native 与 mean-impute 全程打平（60% 时 0.539 vs 0.538）：MCAR 下缺失指示对 v2 也没有额外价值。
  • 值得注意：mean+indicator 在 MCAR 下反而是最差的经典基线——指示列在随机缺失下不含信息，只是白白稀释了 100 个训练样本的自由度。它要到 MNAR 场景才发挥作用（§6）。

真实数据上（对 mini 和 v2 都是分布外，比较更公平），v2 展现了大先验的价值：breast_cancer 上与 mini 打平、wine 上明显更强（60% 缺失时 0.823 vs 0.746）；两个模型的 native 都不输自己的 mean-impute。mini 模型这边，native 与逐任务重训的 logistic 基线大体相当：breast_cancer 上几乎重合，wine（接近线性可分、对线性模型友好）上 logistic 略占优。注意两个数据集上 native 都稳定压过「对模型隐瞒缺失」的 mean-impute——wine 60% 缺失时差距达 5.4pt。

§6 · 结果 · MNAR

信息性缺失：缺失模式变成特征，但必须在预训练里见过

30% 基准缺失率、其中一半列的缺失概率随类别升高（模拟「收入高的人跳过收入题」）。同一批 400 个任务、同样的缺失位置喂给所有策略：

  • cells · native 0.706 —— 比它自己的零缺失基准 0.669 还高 3.7pt。缺失位置携带的标签信息超过了被抹掉的数值信息，且模型学会了在 in-context 中读取它。
  • 结构相同的 vanilla 只有 0.620，甚至输给经典的 mean+indicator（0.653）。缺失指示通道人人都有，「会不会用」完全取决于预训练分布里有没有 MNAR 任务。
  • 所有先填补再建模的流程都摧毁了信号：mean-impute 0.617/0.578、knn 0.590。填补做得越「好」，缺失痕迹越干净，MNAR 信号毁得越彻底。
  • 真实基线一锤定音：TabPFN v2 native 0.635 vs 自身 mean-impute 0.634——差 0.1pt，统计上为零。拥有约 1.3 亿数据集先验、7M 参数的原版模型完全不利用缺失模式（其先验只有 MCAR），在同一批任务上被 1.6M 的 MNAR 预训练模型超出 7.1pt。能力差距源自训练分布，而非规模。

对原版 TabPFN 的推论——现已实测验证：Nature v2 论文的先验只模拟了 MCAR，没有 MNAR。配对实验中 v2 的 native（0.635）与 mean-impute（0.634）完全打平：原版模型确实不利用缺失模式。把 MNAR 模拟加进自研先验（或微调数据）是超越原版的明确机会点——1.6M 的 cells 模型（0.706）在同一批任务上超出 v2 达 7.1pt。

§7 · cells vs vanilla

差别只在训练数据里的缺失长什么样

vanilla 的训练批次

只有先验内建的稀疏随机缺失：每任务缺失率 ∼U[0, 10%]，均匀撒在所有单元格上，与标签无关。

cells 的训练批次

70% 任务被追加腐蚀：任务级缺失率 u∼U[0,0.6]、每行独立抽取缺失率 ∼U[0,2u]（问卷式完整度差异）；其中 30% 任务的一半列缺失概率随类别缩放（MNAR）。

两个模型的前向函数、参数量、先验、步数、种子全部一致；训练曲线也几乎重合（cells 因任务更难，训练准确率略低但更稳）。所有 MNAR 差异（+8.6pt）都来自这一处数据分布的改变。

注意：曲线是各自训练分布上的准确率——vanilla 在干净任务上测、cells 在含重度缺失的任务上测，纵向不可直接比较；可比的是收敛形态与稳定性。

§8 · 结论

四条可执行的结论

  1. 问卷式逐行缺失对 TabPFN 家族是原生输入。把所有问卷对齐到统一问题全集、没答的填 NaN 即可；不要预先填补——mean/KNN 填补在所有实验里都劣于 native NaN。
  2. 随机缺失（MCAR）不需要专门训练。只见过 ≤10% 缺失的模型平滑泛化到 60% 缺失；实测原版 TabPFN v2 同样平滑退化、native 与 mean-impute 打平——处理高比例 MCAR 直接用即可。
  3. 信息性缺失（MNAR）需要专门的先验。这是问卷数据最现实的情形，也是本实验最大的增量发现：预训练先验含 MNAR → 缺失模式从噪声变成特征（0.706 > 0.669），并同时击败经典 indicator 方法与未见过 MNAR 的同结构模型；实测原版 v2 完全不利用缺失模式（0.635 ≈ 0.634），被 1.6M 的 MNAR 预训练模型超出 7.1pt——差距来自训练分布而非规模。
  4. 结构上不需要发明新东西。缺失指示通道（≈ 每个 cell token 的可学习 mask 分量）是 v2 已有且历史悠久的设计；投资应放在训练分布——先验里的缺失机制设计——而非架构。

§9 · 总体评价

mini demo 的价值不在于「比 v2 好」

综合能力上 v2 明显更强；本 demo 的贡献是用严格配对的实验证明了两件事：自研 PFN 路线端到端可行且便宜（笔记本 1 小时、1.6M 参数、真实数据零样本 0.94–0.96），以及 v2 存在一个可以靠「改训练分布」补上的真实能力缺口（MNAR）。生产上不应把 mini 当通用分类器；研究上它的方法论与发现成立。

场景谁强证据该信几分
通用表格分类（真实数据）v2 明显强wine 60% 缺失 0.823 vs 0.746可信——双方都分布外的公平场地，大先验的价值所在
合成任务mini 略高0.669 vs 0.652不算数——任务采自 mini 自己的先验，纯主场优势
MCAR 缺失鲁棒性打平双方 native ≈ mean-impute可信，v2 官方宣称得到独立复核
MNAR（信息性缺失）mini 显著强0.706 vs 0.635（+7.1pt）可信但有保留（见注意点 2）

实际价值

  1. De-risk 了自研路线（方案 D）：全套可复用脚手架已就绪——先验、双轴注意力模型、训练/评估/配对基线 harness，领域专用 PFN 可直接继承。
  2. 找到论文级缺口并给出因果证据：v2 在 MNAR 下 native 与自身 mean-impute 打平 → 确实不利用缺失模式；同结构消融 → 差距完全来自训练分布。
  3. 给出工程决策依据：缺失是随机的 → 直接 v2 + NaN 填充；缺失是信息性的 → 才值得微调或自研。

注意点（诚实清单）

  1. 别脱离「主场优势」标注引用合成任务上 mini > v2 的数字。
  2. MNAR 增益有机制匹配成分：cells 模型训练与评估用同一族 MNAR 机制；对真实问卷的任意跳题模式不保证同样幅度，需在先验里针对性建模。
  3. 利用缺失模式是双刃剑：缺失机制漂移（问卷改版、跳题逻辑变化）会让学到的关联失效甚至反向；生产中应监控缺失分布稳定性。
  4. 有一个未测的廉价方案：给 v2 输入手动拼接 0/1 缺失指示列（logreg+indicator 的技巧用在 TabPFN 上），零训练成本，值得先试。
  5. 规模保留：mini 先验很窄（≤10 特征、≤4 类、≤160 行、纯数值）；MCAR 下 cells 与 vanilla 的 ~0.3pt 差异在噪声内。
  6. 许可红线：商用相关的生成/蒸馏/对比发布只能用 v2 权重。

下一步（按性价比排序）

  1. 在真实问卷数据上跑 v2 + NaN 填充与 v2 + 手动指示列，判断真实缺失是否信息性；
  2. 若信息性且指示列不够 → 官方 finetuning/ 模块 + 缺失机制模拟做轻量微调（方案 C）；
  3. 领域结构超出 v2 先验覆盖时才升级到完整自研（方案 D）——基础设施已在 minipfn/ 中。
